# P2P StreamLink v2.0

Local dashboard for searching, buffering, and instantly streaming P2P media through VLC ‚Äî with a Mullvad VPN kill-switch.

---

## Quick Start

### 1 ‚Äî Install prerequisites on this machine

| Tool | macOS default path | Notes |
|------|--------------------|-------|
| [VLC](https://videolan.org/vlc/) | `/Applications/VLC.app` | Media player ‚Äî must be local |
| [qBittorrent](https://qbittorrent.org/) | `/Applications/qbittorrent.app` | Download engine ‚Äî must be local |
| [Mullvad VPN](https://mullvad.net/) | App + CLI | Kill-switch guard ‚Äî must be local |
| [Jackett](https://github.com/Jackett/Jackett/releases) | `.app` or extracted folder | P2P indexer ‚Äî **can be on a remote PC** |

> `setup.py` auto-detects all four at their default install paths.

**Jackett on a separate machine?**  
Skip the Jackett install step above. During `setup.py` you will be prompted for the Jackett URL ‚Äî enter your remote machine's address (e.g. `http://192.168.1.50:9117`). `run.py` will check reachability but will not try to launch it locally.

### 2 ‚Äî Run setup (once)

```bash
python3 setup.py
# or: make setup
```

This will:
- Create a `.venv` and install all Python packages
- Auto-detect VLC, qBittorrent, Mullvad, and Jackett
- Prompt for passwords, download path, and the Jackett URL (accepts remote addresses)
- **Write the qBittorrent Web UI config** directly to its ini file (port 8081, localhost auth disabled, CSRF off)
- Generate your `.env` file

> Run setup with qBittorrent **closed** so it picks up the new config on next launch.  
> This step configures qBittorrent on **this machine only** ‚Äî if qBittorrent is elsewhere, configure its Web UI manually.

### 3 ‚Äî Start everything

```bash
python3 run.py
# or: make run
```

This will:
- **VLC** ‚Äî start with Lua HTTP interface on port 8080 (restarts VLC if already open without HTTP)
- **qBittorrent** ‚Äî start if not running, wait for Web UI on port 8081
- **Jackett** ‚Äî if `INDEXER_URL` is localhost: find and launch the binary; if remote: check reachability only
- **Mullvad** ‚Äî verify connected; prompts to continue if not
- Open the dashboard in your browser at **http://127.0.0.1:7000**

Press **Ctrl+C** to stop the dashboard. VLC, qBittorrent, and Jackett keep running.

## Reboot & Scheduled Restart (Jackett fix)

The most reliable cure for some wedged-Jackett states is a full machine reboot. The admin **System** tab has two controls for this:

- **Reboot Machine** ‚Äî restarts the whole host immediately (not just the web server).
- **Scheduled Restart** ‚Äî reboots the host daily at a set time (e.g. `00:00` Pacific), but only when idle: at the scheduled time it checks whether anything was used in the last *N* minutes (default 15). If idle, it reboots; if in use (active playback, a stream, or a download), it waits *N* minutes and re-checks, repeating until the box is free.

### ‚öÝÔ∏è Make the server come back on its own

A reboot only helps if the host returns to a running StreamLink afterward. That requires **two** things on the host:

1. **Configure OS auto-login** so the user account logs in automatically after a reboot ‚Äî no password prompt at the lock screen. The launchd / systemd user agent and Windows logon task only start once that user session exists.
   - **macOS:** System Settings ‚Üí Users & Groups ‚Üí *Automatically log in as‚Ä¶* (requires FileVault to be off).
   - **Windows:** run `netplwiz`, untick *Users must enter a user name and password‚Ä¶*, or use a tool like Sysinternals **Autologon**.
   - **Linux:** enable autologin in your display manager (e.g. GDM `AutomaticLoginEnable=true`, or `loginctl enable-linger $USER` for a headless systemd user service).
2. **Install the StreamLink system service** so it relaunches on login/boot:
   ```bash
   python3 run.py --install     # registers launchd / systemd / Task Scheduler
   python3 run.py --status      # confirm it's registered
   ```

Without both, a scheduled or manual reboot will take the dashboard offline until you re-run `run.py` by hand.

## Configuration

All settings live in `.env` (generated by `setup.py`). See `.env.example` for the full list with descriptions. Key variables:

| Variable | Default | Notes |
|----------|---------|-------|
| `INDEXER_URL` | `http://localhost:9117` | Jackett ‚Äî accepts remote `http://host:port` |
| `INDEXER_API_KEY` | _(empty)_ | Paste from Jackett dashboard |
| `QBIT_URL` | `http://localhost:8081` | qBittorrent Web UI |
| `VLC_URL` | `http://localhost:8080` | VLC Lua HTTP interface |
| `VLC_PASSWORD` | `vlcpassword` | Must match what VLC is launched with |
| `QBIT_DOWNLOAD_PATH` | `~/Downloads/StreamLink` | Where files are saved (created automatically) |
| `BUFFER_MIN_MB` | `15.0` | Start VLC once this many MB are downloaded |
| `BUFFER_MIN_PCT` | `1.0` | ‚Ä¶or once this % of the file is downloaded |
| `INDEXER_CATEGORIES` | `0` | `0` = all categories (no filter); `2000` = Movies, `5000` = TV |

## How It Works

```
Search ‚Üí Jackett API ‚Üí results sorted by seeders
  ‚Üì
Play ‚Üí qBittorrent adds magnet ‚Üí sequential download enabled
  ‚Üì
Buffer check (async poll, never time.sleep) ‚Üí 15 MB or 1% threshold crossed
  ‚Üì
VLC ‚Üê file:/// URI sent to Lua HTTP interface ‚Üí playback begins
  ‚Üì
Stop ‚Üí torrent + local file deleted from qBittorrent ‚Üí VLC stopped
```

### VPN Kill-Switch

Every 3 seconds the backend runs `mullvad status`. If `"Connected"` is absent:

1. `qbittorrent` / `qbittorrent.exe` is killed via `psutil`
2. A full-screen red warning overlays the dashboard
3. All new `/api/stream` requests return HTTP 403 until VPN reconnects

## API Reference

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/api/search?q=‚Ä¶` | Search Jackett, returns results sorted by seeders |
| `POST` | `/api/stream` | Start stream pipeline `{"magnet": "‚Ä¶", "title": "‚Ä¶"}` |
| `POST` | `/api/stop` | Cancel pipeline, delete torrent + file, stop VLC |
| `GET`  | `/api/state` | Full state snapshot (JSON) |
| `GET`  | `/api/events` | SSE stream ‚Äî events: `state`, `vpn_status`, `stream_status`, `library_update`, `progress_saved` |

| `POST` | `/api/vlc/seek?delta=N` | Seek relative to current position (seconds; negative = rewind) |
| `POST` | `/api/vlc/seek/to?position_pct=N` | Seek to absolute position (0‚Äì100 %) |
| `GET`  | `/api/vlc/tracks` | Audio and subtitle track list with current selections |
| `POST` | `/api/vlc/track/audio/{id}` | Switch audio track by ES ID |
| `POST` | `/api/vlc/track/subtitle/{id}` | Switch subtitle track by ES ID (‚àí1 = off) |
| `GET`  | `/api/library` | List library items (add `?profile_id=‚Ä¶` for watch progress) |
| `POST` | `/api/library/download` | Add item to library `{"magnet","title","series","season","episode","save_path"}` |
| `GET`  | `/api/library/{id}/files` | File list for a multi-episode item |
| `POST` | `/api/library/{id}/play` | Play item(s) in VLC `{"profile_id","files":[],"seek_first_to":null}` |

| `GET`  | `/api/profiles` | List profiles |
| `POST` | `/api/profiles` | Create profile `{"name","color"}` |
| `DELETE` | `/api/profiles/{id}` | Delete profile and its watch history |

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `Indexer unreachable` | Jackett not running; or wrong `INDEXER_URL` in `.env`; or missing API key |
| Remote Jackett not reachable | Confirm Jackett is running on the other PC and its port is reachable from this machine |
| qBittorrent Web UI won't respond | Close qBit, run `setup.py` again, relaunch |
| VLC doesn't play the file | Check VLC is open (run.py starts it); confirm `VLC_PASSWORD` in `.env` matches |
| `mullvad CLI not found` | Add Mullvad CLI to PATH, then re-run `setup.py` |
| Port conflict on 8080 or 8081 | VLC defaults to 8080, qBittorrent to 8081 ‚Äî **they must be different**. `setup.py` warns if they collide. Edit `VLC_URL` / `QBIT_URL` in `.env` and restart the affected service. For qBittorrent also update its Web UI Preferences. |
| Dashboard shows "VPN DISCONNECTED" | Reconnect Mullvad ‚Äî the overlay clears automatically once reconnected |
