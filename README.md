# P2P StreamLink v2.0

Local dashboard for searching, buffering, and instantly streaming P2P media through VLC — with a Mullvad VPN kill-switch.

---

## Quick Start

### 1 — Install prerequisites on this machine

| Tool | macOS default path | Notes |
|------|--------------------|-------|
| [VLC](https://videolan.org/vlc/) | `/Applications/VLC.app` | Media player — must be local |
| [qBittorrent](https://qbittorrent.org/) | `/Applications/qbittorrent.app` | Download engine — must be local |
| [Mullvad VPN](https://mullvad.net/) | App + CLI | Kill-switch guard — must be local |
| [Jackett](https://github.com/Jackett/Jackett/releases) | `.app` or extracted folder | P2P indexer — **can be on a remote PC** |

> `setup.py` auto-detects all four at their default install paths.

**Jackett on a separate machine?**  
Skip the Jackett install step above. During `setup.py` you will be prompted for the Jackett URL — enter your remote machine's address (e.g. `http://192.168.1.50:9117`). `run.py` will check reachability but will not try to launch it locally.

### 2 — Run setup (once)

```bash
python3 setup.py
# or: make setup
```

This will:
- Create a `.venv` and install all Python packages
- Auto-detect VLC, qBittorrent, Mullvad, and Jackett
- Prompt for passwords, download path, and the Jackett URL (accepts remote addresses)
- **Write the qBittorrent Web UI config** directly to its ini file (port 8081, localhost auth disabled, CSRF off)
- Generate your `.env` file

> Run setup with qBittorrent **closed** so it picks up the new config on next launch.  
> This step configures qBittorrent on **this machine only** — if qBittorrent is elsewhere, configure its Web UI manually.

### 3 — Start everything

```bash
python3 run.py
# or: make run
```

This will:
- **VLC** — start with Lua HTTP interface on port 8080 (restarts VLC if already open without HTTP)
- **qBittorrent** — start if not running, wait for Web UI on port 8081
- **Jackett** — if `INDEXER_URL` is localhost: find and launch the binary; if remote: check reachability only
- **Mullvad** — verify connected; prompts to continue if not
- Open the dashboard in your browser at **http://127.0.0.1:7000**

Press **Ctrl+C** to stop the dashboard. VLC, qBittorrent, and Jackett keep running.

---

## After First Run

- Open Jackett at the URL you configured (default `http://127.0.0.1:9117`) and add indexers for public domain content
- Copy the Jackett API key into `.env` → `INDEXER_API_KEY=…`
- Re-run `python3 setup.py` at any time to change settings — it merges into the existing config, never overwrites from scratch

## Configuration

All settings live in `.env` (generated by `setup.py`). See `.env.example` for the full list with descriptions. Key variables:

| Variable | Default | Notes |
|----------|---------|-------|
| `INDEXER_URL` | `http://localhost:9117` | Jackett — accepts remote `http://host:port` |
| `INDEXER_API_KEY` | _(empty)_ | Paste from Jackett dashboard |
| `QBIT_URL` | `http://localhost:8081` | qBittorrent Web UI |
| `VLC_URL` | `http://localhost:8080` | VLC Lua HTTP interface |
| `VLC_PASSWORD` | `vlcpassword` | Must match what VLC is launched with |
| `QBIT_DOWNLOAD_PATH` | `~/Downloads/StreamLink` | Where files are saved (created automatically) |
| `BUFFER_MIN_MB` | `15.0` | Start VLC once this many MB are downloaded |
| `BUFFER_MIN_PCT` | `1.0` | …or once this % of the file is downloaded |
| `INDEXER_CATEGORIES` | `0` | `0` = all categories (no filter); `2000` = Movies, `5000` = TV |

## How It Works

```
Search → Jackett API → results sorted by seeders
  ↓
Play → qBittorrent adds magnet → sequential + first/last piece priority forced
  ↓
Buffer check (async poll, never time.sleep) → 15 MB or 1% threshold crossed
  ↓
VLC ← file:/// URI sent to Lua HTTP interface → playback begins
  ↓
Stop → torrent + local file deleted from qBittorrent → VLC stopped
```

### VPN Kill-Switch

Every 3 seconds the backend runs `mullvad status`. If `"Connected"` is absent:

1. `qbittorrent` / `qbittorrent.exe` is killed via `psutil`
2. A full-screen red warning overlays the dashboard
3. All new `/api/stream` requests return HTTP 403 until VPN reconnects

## API Reference

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/api/search?q=…` | Search Jackett, returns results sorted by seeders |
| `POST` | `/api/stream` | Start stream pipeline `{"magnet": "…", "title": "…"}` |
| `POST` | `/api/stop` | Cancel pipeline, delete torrent + file, stop VLC |
| `POST` | `/api/vlc/pause` | Toggle VLC pause/resume |
| `POST` | `/api/vlc/volume/up` | Volume +10% |
| `POST` | `/api/vlc/volume/down` | Volume −10% |
| `GET`  | `/api/state` | Full state snapshot (JSON) |
| `GET`  | `/api/events` | SSE stream — events: `state`, `vpn_status`, `stream_status` |

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `Indexer unreachable` | Jackett not running; or wrong `INDEXER_URL` in `.env`; or missing API key |
| Remote Jackett not reachable | Confirm Jackett is running on the other PC and its port is reachable from this machine |
| qBittorrent Web UI won't respond | Close qBit, run `setup.py` again, relaunch |
| VLC doesn't play the file | Check VLC is open (run.py starts it); confirm `VLC_PASSWORD` in `.env` matches |
| `mullvad CLI not found` | Add Mullvad CLI to PATH, then re-run `setup.py` |
| Port conflict on 8080 or 8081 | VLC defaults to 8080, qBittorrent to 8081 — **they must be different**. `setup.py` warns if they collide. Edit `VLC_URL` / `QBIT_URL` in `.env` and restart the affected service. For qBittorrent also update its Web UI Preferences. |
| Dashboard shows "VPN DISCONNECTED" | Reconnect Mullvad — the overlay clears automatically once reconnected |
