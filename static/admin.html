When a video is played on the TV, immediately prep that episode for on-device streaming, then the rest of the playlist one episode at a time. Subtitles

One preferred language drives online search, automatic track selection on playback, and AI generation. StreamLink tells VLC explicitly whether subs are on or off, so they never sneak on when they should be off. Each user can override the on/off default in their own Profile Settings.

Preferred when selecting a track and when searching online. Also the target for AI generation (whisper can only translate to English; other targets transcribe the spoken language).

Subtitles On By Default
Whether playback starts with subtitles on. Users can override per-profile.
Auto-Search Online
When subs are on and no preferred-language track is embedded, fetch one from OpenSubtitles on playback. Falls back to AI subs if the search comes up empty.
Upgrade To Real Subs When Found
Release subtitles often finish downloading after the video, so an AI sub may be applied first. When a real preferred-language sub turns up, switch to it automatically and notify the player.
Assume A Lone Subtitle Is Correct
When exactly one subtitle is found and its language can't be read from the filename, treat it as the preferred language instead of skipping it.

Auto-Generated Subtitles (AI)

Transcribes audio into a subtitle track for files that ship none usable — during stream-prep, or on demand from the player. Uses whisper.cpp on the host (slower than real-time; best left to when the box is idle).

whisper.cpp or its model isn't installed on this host — generation is disabled. Re-run setup.py to add it.

Doubles transcription time for foreign-language audio. The target language is set above in the Subtitles card.

Optional Components

Install the portable dependencies the auto-updater can't fetch on its own — ffmpeg, fpcalc, and the whisper.cpp engine + model for AI subtitles. Downloads run on the host and persist across updates. ffmpeg / whisper.cpp binaries are Windows-only here; on Linux/macOS use your package manager.

Auto Update

Periodically pulls the chosen branch from GitHub. When a new commit lands, it runs setup.py (non-interactive), regenerates the supervisor wrapper script (no UAC — just a file write), then reboots the host for a clean restart on the new code. Auto-apply waits until the host is idle so a routine update can't interrupt a stream.

Only main / beta / alpha may be selected.

Developer mode. Not all branches are functional — switching to one may corrupt or wipe your data and require a full manual reinstall of StreamLink to recover. Only use this if you know what you're doing.

Checks out an exact build by SHA (7–40 hex chars) as a detached HEAD. Auto-update is disabled while pinned — switch back to a branch to re-enable it.

Between 1 and 168 (1 week).

When off, the dashboard shows a banner and the admin clicks Apply Now to update. Auto-apply waits until the host is idle so it never interrupts a stream.
